# Doctor JSON v1

`runforge doctor --repo /path --working-directory frontend --format json` is read-only and returns `ready`, `ready_with_warnings`, or `blocked`. It reports the canonical repository root and execution root separately, plus the execution-root package manager, lockfile, and validation commands. Dirty and detached worktrees are warnings; missing targets, non-Git or unborn targets, an invalid/escaping working directory, missing explicitly required runtime tools, and unsafe artifact placement are blockers. Dependency preparation is blocking only when the selected strategy is `required`; `if-needed`, `disabled`, and `reuse-existing` are evaluated during bounded execution. Docker is required only for `--runtime docker`; when requested, doctor verifies the daemon and selected local image (`runforge:local` by default). GitHub is checked only for `--publication draft-pr`. DB, production, and secrets are never required for basic operation.
